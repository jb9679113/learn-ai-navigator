# 任务：生成一个完整的 AI 学习资源导航网站项目

## 项目描述
我需要一个现代化的资源导航网站，用于集中管理 AI 学习相关的网站、GitHub 仓库、文档等资源。用户是我自己（管理员）以及未来的培训学员（访客）。项目必须包含前端、后端、数据库，可直接运行。

## 已有资源示例（作为种子数据）
1. Vibe coding学习项目: https://github.com/2025Emma/vibe-coding-cn
2. easy vibe学习网站: https://datawhalechina.github.io/easy-vibe/zh-cn/
3. TRAE官方知识库: https://lcnziv86vkx6.feishu.cn/wiki/XZOSwI51wi5a5okxCF4cAxHSnBh
4. AI提示词项目: https://github.com/jb9679113/prompts.chat

以后我会继续添加更多资源，因此需要一个后台管理界面来增删改资源。

## 技术要求
- 前端框架：React + Next.js (App Router) + TypeScript
- 样式：TailwindCSS
- 后端：Next.js API Routes
- ORM：Prisma
- 数据库：SQLite（开发环境），且提供迁移到 PostgreSQL 的配置
- 认证：后台管理使用简单的密码保护（读取环境变量 `ADMIN_SECRET`，默认为 "admin123"）
- 部署准备：所有配置适合部署到 Vercel

## 功能要求

### 前台（公开访问）
1. **首页**：显示 Hero、热门分类入口（取自现有分类）、最新添加的 6 个资源卡片、推荐资源（按 rating 排序前 3）。
2. **资源列表页** (`/resources`)：
   - 左侧树形分类菜单（支持无限级，基于 parentId），可展开折叠。
   - 右侧资源卡片列表（显示名称、简介、标签、难度标识、来源类型图标）。
   - 顶部有搜索框（按标题、简介、标签模糊匹配）和筛选器（难度、标签、来源类型）。
   - 卡片点击后弹出详情弹窗（显示完整描述、链接按钮、我的笔记区域）。
3. **资源详情独立页**（可选，但最好实现 `/resource/[id]` 作为 SEO 友好页面）。
4. **我的笔记**：在前台的资源详情弹窗中，管理员可以看到并编辑“我的笔记”字段（仅管理员可见，访客不显示）。如果未登录，访客看不到笔记区域。

### 后台管理（`/admin`）
- 通过简单密码访问。
- 界面包括：
  - 资源列表（表格形式，带编辑/删除按钮）。
  - 添加/编辑资源表单（字段：名称、网址、描述、分类（下拉树）、标签（多选或输入）、难度、评分、来源类型、我的笔记）。
  - 分类管理（树形列表，可增删改，支持拖动排序？简单起见，提供添加/编辑/删除即可）。
  - 标签管理（简单列表，增删改）。
- 所有操作通过 API 完成，使用 Prisma 操作数据库。

### 数据模型（Prisma Schema）
需要定义以下模型：
- Category (id, name, parentId, order)
- Resource (id, name, url, description, categoryId, tags(JSON或关联表?), difficulty, rating, sourceType, myNotes, isPublished, createdAt, updatedAt)
- 标签可用简单方式：Resource 表中加 tags Text[]（SQLite 支持数组）。
- 暂不需要用户表（后续可扩展，但本次先不做学员登录和进度记录）。

### 其他要求
- 种子脚本：在首次运行时自动创建示例分类（如“Vibe Coding”、“AI 提示词”、“官方文档”等）并将给出的 4 个资源插入数据库。
- UI 美观：使用 TailwindCSS 制作卡片风格，响应式设计，支持移动端。
- 代码结构清晰，注释充分，每个组件和 API 路由都有基本说明。
- 提供 README.md，包含：如何安装、设置环境变量、运行种子数据、构建部署等步骤。
